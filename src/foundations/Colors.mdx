# Colors

All colors are CSS custom properties defined in `src/styles/tokens.css`,
under `:root` (dark, default) and `:root.light`. Never hardcode a hex value
in a component — reference the token (`var(--accent)` in plain CSS, or the
`accent` / `bg` / `text-dim` etc. Tailwind utilities that `globals.css` maps
to them) so switching themes or rebranding a copy of this system is a
token-file edit, not a find-and-replace.

Toggle the **Theme** control in the toolbar above to see how these repaint.

## Light

itself shifts (teal → blue) and surfaces move from near-black to a cool
off-white, matching the source brand exactly.

## Usage rules

- **Text hierarchy**: `text` for headings and primary content, `text-dim`
  for body copy, `text-muted` for captions/timestamps/disabled text. Don't
  skip a level (e.g. heading in `text-muted`).
- **Accent is for one thing at a time**: the primary CTA on a screen, the
  active tab/nav item, a focus state, a link on hover. If two elements need
  to compete for attention, at most one gets the accent.
- **`-faint` variants are fills, not text.** Use `accent-faint` /
  `danger-faint` etc. as a tinted background behind accent/danger text or
  icons (badges, alert panels, icon chips) — never as the text color itself,
  it fails contrast.
- **Feedback colors map 1:1 to intent**: success = completed/positive,
  warning = needs attention, danger = destructive/error, info = neutral
  informational. Don't reach for `danger` because red "looks stronger."
